\doxysubsection{Detailed Description}
Class for connecting \char`\"{}\+Thermal Capture Grabber USB\char`\"{}. 

Connecting Thermal Capture Grabber USB needs three steps\+:
\begin{DoxyEnumerate}
\item Include the library header file \char`\"{}thermalgrabber.\+h\char`\"{} into your project.
\item Write a callback function for receiving Tau\+Raw\+Bitmaps.
\item Use the constructor of \mbox{\hyperlink{class_thermal_grabber}{Thermal\+Grabber}}. You will have to put in 2 parameters. Firstly put in the pointer of the callback function. Secondly add the pointer of the calling instance in the constructor. 
\end{DoxyEnumerate}

Change the watchdog count reset value. 

The watchdog checks every second for frame activity (frames from TCGrabber\+USB). Standardly the counter is set to 10 seconds. If the watchdog doesn\textquotesingle{}t notice any transfers in that time libthermalgrabber tries to reconnect TCGrabber\+USB. Normally the counter shouldn\textquotesingle{}t be modified but sometimes it\textquotesingle{}s necessary (e.\+g. when using the external sync input -\/ low-\/high-\/transition -\/ on pin5 of TCGrabber\+USB with single frame transfers -\/ the watchdog might trigger between two sync input signals). Please keep in mind that there are situations where TCGrabber\+USB can\textquotesingle{}t send frames for a couple of seconds because \mbox{\hyperlink{class_tau}{Tau}} 2 Core doesn\textquotesingle{}t transfers data (e.\+g. while ffc-\//shutter-\/events). By using \char`\"{}0\char`\"{} for parameter wait\+\_\+time the watchdog becomes inactive (will keep inactive until next connect).

{\footnotesize\ttfamily \mbox{\hyperlink{libusb_8h_a8ca3594d1761da8da91be2851f4e8f21}{void}} Thermal\+Grabber\+::enable\+External\+Sync\+As\+Slave (\begin{DoxyParamCaption}{ }\end{DoxyParamCaption})}



Enable synchronization mode as slave. 

Requirements\+:
\begin{DoxyEnumerate}
\item This function is supported since firmware 1.\+3 of TCGrabber\+USB (as of december 2017). Older firmware versions will ignore the TCGrabber\+USB related setting of this command. But \mbox{\hyperlink{class_tau}{Tau}} 2 Core setting of EXTERNAL\+\_\+\+SYNC will be done. Accordingly \mbox{\hyperlink{class_tau}{Tau}} 2 Core will not work with an TCGrabber\+USB with older firmware!
\item Requires \mbox{\hyperlink{class_tau}{Tau}} 2 Core with fast video core (25Hz/30\+Hz)
\end{DoxyEnumerate}

The frame rate of the \mbox{\hyperlink{class_tau}{Tau}} 2 Core depends on the synchronization signal. Normally this signal is generated by the \mbox{\hyperlink{class_tau}{Tau}} 2 Core itself but can also be applied from outside (e.\+g. from TCGrabber\+USB or pin5 (external sync input)). In this case the sync signal has to be fed in to pin5 (external sync input) of TCGrabber\+USB. The sync signal is then forwarded to \mbox{\hyperlink{class_tau}{Tau}} 2 Core and to pin6 (external sync output) of TCGrabber\+USB. Please take care to only use frequencies in the allowed spectrum. The allowed frequency spectrum depends on whether the PAL or NTSC setting of \mbox{\hyperlink{class_tau}{Tau}} 2 Core is active.
